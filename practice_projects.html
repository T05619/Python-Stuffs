Practice Projects

On this page you can find a list of practice projects which you can attempt making in python! Each project has requirements however you do not need to complete all the requirements and can make the project your own, you will also need to make certain bits of information up. Some of the projects will be harder than others. If you need help DM me on discord!

ELECTRIC CAR CHARGING STATIONS

Create a smart parking system for electric vehicles that allows users to reserve parking spaces with charging stations.

Requirements:

  • There should be 5 different stations with 5 charging ports at each
  • They should have to sign into an account/create a account
  • When they book a station it should "charge" printing a bill
  • After booking a station the program should end

EXPENSE TRACKER

Input cost of each purchase and what it was then it is totalled.

Requirements:

  • Create a account/sign in to a account
  • Show the total they have already spent (and entered into the program before) on log in
  • Take input of what they bought with its cost and save it
  • Print the total after they have just bought the item
  • Option to see list of everything they bought and what it costed

CUSTOM RECIPE BOOK

User can create recipes and delete them.

Requirements:

  • Get title for a recipe and then what they recipe is in steps (maybe ask for number of steps before)
  • After creating a recipe, they have the option to create a new recipe or view one they have made
  • If they chose to view a recipe you must print it out in the correct order

TASK MANAGER

User can create tasks that they can mark as working on or complete and start as not completed.

Requirements:

  • Get the user to enter task
  • Give option to change status of a task or create a new one
  • Option to show all tasks and their status

INVENTORY MANAGEMENT SYSTEM

Stock system where you can view all the stock of items.

Requirements:

  • Save the stock when they create new stock/sell some
  • Option to create new item (will take a price, name and stock amount)
  • Option to view all stock
  • Option to update stock where they will enter what items where sold and the amount that was sold
  • Option to show total money made with the amount of each items sold (Item, Amount, Money made)
  • Add stock so if there is new stock it will be added to the old stock

CONTACT BOOK

Users can sign/create an account and add, edit and delete contacts.

Requirements:

  • Get the user to create an account/login
  • Option to create new contact edit old one or delete one
  • Use dictionaries to store the information

GRADE BOOK

Teachers can edit students grades and students can view their subjects.

Requirements:

  • Sign in system where if a teacher signs in (from a list of teachers) they have the ability to add new students with the subjects that they take and then add the grades that they have in their subjects.
  • Teachers should also be able to edit students grades.
  • You must save the grades information.
  • If a student signs in that is registered with grades it will show them their subjects the grade they have in that subject and if they are currently passing or not.
